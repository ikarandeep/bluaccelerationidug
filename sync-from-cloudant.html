Sync from Cloudant

Roger, the data analyst at a food product company, would like to analyze USDA Food and vitamin data freely available from the web for creating new business opportunity for his food products company. He manages his NoSQL data in IBM Cloudant database and wants to leverage BLU Acceleration for Cloud for statistical analysis or reporting purposes.

In this scenario, you will be using US department of Agriculture Food database.
You have a choice of downloading the JSON data from http://ashleyw.co.uk/files/foods-2011-10-03.json.zip and uploading it to a Cloudant database at https://cloudant.com OR, for the purpose of this workshop, we have published this dataset for IDUG attendees here: https://idug.cloudant.com/food

Creating the Sync

1 Login to the BLU Acceleration for Cloud web console. Click on Manage → Sync from Cloudant
2 Enter the following URL in to Cloudant DB URL: https://idug.cloudant.com/food

If you are using your own database, remember to enter you Username and Password as well.

3 Enter a table name of your choice in Target Table name
4 Click Start Sync. You should see the sync in progress:
5 Refresh the screen and you should see status Running
6 You can go to Manage → Work with Tables and find your schema name and browse the table that you just created for the Cloudant sync
7 You should be able to browse data as it comes in
8 Click Manage → Run Query and feel free to build the query using Select Tables and Columns or type your own query.
9 You can bring the data back to Excel using ODC scenario for further analytic processing OR follow any other scenario such as Analyze with R or Analyze with Cognos on this data.
