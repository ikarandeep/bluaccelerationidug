Bring public datasets from S3

Kelly, the data analyst at Public Policy Institute, wants to be able to load publicly available data sets directly from AWS S3 object storage to BLU Acceleration for Cloud and use built-in analytic features to analyze data.

Generating the Schema

1 Login with the provided user name/password
2 Click Manage → Work with Tables from the main menu
3 Click on the + button and the DDL statement editor will open on the right.
4 Download the DDL file here: censusPopulation_new.ddl.txt
5 Copy-Paste the DDL statements for the tables that you want to create and click on the Run DDL button. Users can also create all of the tables from the attached file at once

Loading Data

6 Click Manage → Load Data from the main menu
7 Open Schedule a load section
8 Choose Amazon S3 → Browse public bucket and provide the bucket name, e.g. ibm.idug.public. Then, click on the Browse S3 Files button.
9 Select the data file that you want to load as below. Try these data files:
  • Ancestry.csv can be selected for loading data into Ancestry table
  • ss12pca.csv or person_2_5gb.del can be selected for loading data into Person table
Click OK
10 Separator character and date/time options can be set as below. Click Next
11 Select Schema which will be same as your user name as in step 1, also select the table that you are interested in loading
12 Choose either of the options below at Select a table step
13 Select date and time when you want to run this load job, click on Next, Finish and close the alert which appears after the load job gets saved successfully
14 Feel free to follow other analytic scenarios on this data. Browse R samples to get an idea of what you could do with it.
