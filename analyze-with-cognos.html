Analyze with Cognos

Adele, the business analyst at ACME Corporation, wants to create reports using BLU Acceleration and Cognos Business Insight on her sales data. She first would like to get acquainted with the sample reports provided with BLU Acceleration for cloud and then use this scenario to create a report.

Note: Adele needs admin access to be able to perform publishing of Cube. Please ask me so that I can assign you admin privilege. You also need to download and install the following Cognos client software on your laptop. Follow Setup -> Download page. Please ignore installing fixpack and interim fixpack outlined in 2 and 3. Please install Cube designer for this scenario. See below:

1) Please follow Analyze -> Cognos Business Intelligence to browse sample Cognos reports.

2) Launch IBM Cognos Cube Designer (You should have configured your “IBM Cognos Configuration” to point to the Cognos server by following step 4 in above section).

3) Click “Create New from Metadata”.

4) Click OK on the “Invalid Certificate” window.

5) Login with your username and password.

6) Select schema “DB2INST1” from BLUDB, and click OK.

7) Expand BLUDB → DB2INST1 → Tables to view the table list under schema “DB2INST1”.

8) Scroll down, find table “PERSON”, right click the table and select Generate → Cube with dimensions using data sampling.

9) You can rename the generated cube name as you like.

10) Right click the cube and click "Pulish".

11) Select all options on the “Publish” window and click OK (You can rename your package as you like).

12) Get Deploy Results

13) Launch IBM Cognos Connection home page, select “My Folders” tab and check that the published cube is listed.

14) Launch Report Studio.

15) Select the published package under “My Folders”.

16) Select Create new, and then click “Chart”.

17) Select the “Stacked column” type.

18) Pause the pointer over the query explorer button and click Queries.

19) Drag SQL from the Toolbox to the right of Query1.

20) Select the SQL. In the Properties pane, double-click the Data Source property and select BLUDB as data source.

21) Double-click the SQL property and copy paste the SQL below:

select t1."EDU_DESC",t2."GEN_DESC",COUNT(f0."SERIALNO" CONCAT f0."SPORDER"),COUNT(f0."SERIALNO") from "PERSON" f0,"HISPANIC_ORIGIN" t15,"RACE_CODE" t16,"LANGUAGE" t10,"GENDER" t2,"ANCESTRY" t11,"RACE_CODE3" t12,"RACE_CODE2" t14,"COW" t5,"MARITAL_STATUS" t6,"EDUCATION" t1,"CITSTATUS" t9,"TRANSPORTATION" t13,"MILITARY_STATUS" t3,"STATECODE" t8,"PLACE_OF_WORK" t4,"PLACE_OF_BIRTH" t7 where f0."HISP"=t15."HISP_CODE" and f0."RAC1P"=t16."RACE_CODE" and f0."LANX"=t10."LANGUAGE_CODE" and f0."SEX"=t2."GEN_CODE" and f0."ANC1P"=t11."ANC_CODE" and f0."RAC3P"=t12."RACE_CODE" and f0."RAC2P"=t14."RACE_CODE" and f0."COW"=t5."COW_CODE" and f0."MAR"=t6."MAR_CODE" and f0."SCHL"=t1."EDU_CODE" and f0."CIT"=t9."CIT_CODE" and f0."JWTR"=t13."JWTR_CODE" and f0."MIL"=t3."MIL_CODE" and f0."ST"=t8."STATE_CODE" and f0."POWSP"=t4."POWSP_CODE" and f0."POBP"=t7."POBP_CODE" group by t1."EDU_DESC",t2."GEN_DESC"

22) Click Validate to check for errors.

23) Select the Query. In the Properties pane, set the Processing property to Limited Local.

24) Pause the pointer over the page explorer button and click a report page.

25) Drag the column from the query to corresponding axis.

26) Please note, for this chart to display Education Level by Gender, we put <#EDU_DESC#> under <#GEN_DESC#> for x-axis, so that Education Level will be treated as a sub-category for Gender Information.

27) For Default measure (y-axis), we will need specify Aggregate Function and Rollup Aggregation Function. For our case, specify Total for both.

28) From Page Explorer, right click the report page we have been working on, and select Run Page.
